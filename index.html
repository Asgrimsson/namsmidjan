Hvernig sinkar þetta saman?

Netlify hýsir vefinn og AI function; Supabase geymir gögnin; vafrinn heldur tímabundnu local afriti.

Flæðið: Kennari býr til verkefni → sendir local gögn í Supabase → nemendur opna með kóða → kennari sækir nýjustu gögn í bekkjayfirliti.

Leiðbeiningar

Stuttar kennaraleiðbeiningar fyrir fyrstu skólastofuprófun.

Fyrsta kennsluprófun

  1. Skráðu þig inn sem kennari.
  2. Búðu til verkefni með AI.
  3. Vistaðu sem verkefnapakka.
  4. Sendu local gögn í Supabase.
  5. Skrifaðu verkefnakóðann á töfluna.

Nemendur

  1. Opna vefinn.
  2. Smella á Nemendagátt.
  3. Skrifa nafn eða hóp.
  4. Skrifa verkefnakóða.
  5. Merkja stöðu.

Sync-reglan

Áður en tími byrjar: Sækja gögn úr Supabase.

Eftir breytingar: Senda local gögn í Supabase.

Eftir vinnu nemenda: Sækja nýjustu gögn í bekkjayfirliti.

Ef eitthvað virkar ekki

  • Athugaðu kennarainnskráningu.
  • Prófaðu tengingu í Gagnagrunnur.
  • Athugaðu Netlify Function ef AI svarar ekki.
  • Endurhlaðaðu síðuna eftir uppfærslu.

Kennarasvæði

Hér getur þú bætt við námsvef. Í v0.2 vistast breytingar í vafranum með localStorage. Síðar tengjum við þetta við Supabase svo gögnin vistist miðlægt.
Hér býrð þú til verkefnaspjöld. Þau birtast í Verkefnabankanum og nemendur geta fundið þau með kóða eins og 5B eða 5B-PROSENTUR.
Hér skráir kennari sig inn með Supabase Auth. Í v0.5 þarf kennari að vera skráður inn til að senda eða breyta námsvefjum og verkefnum í Supabase. Nemendur geta áfram notað kóða í Nemendagátt til að skila framvindu.

Hér tengir þú Námsmiðjuna við Supabase. Fyrir v0.5 skaltu keyra supabase_v0_5_security.sql í Supabase SQL Editor eftir v0.4 schema. Settu síðan inn Project URL og publishable/anon key.
